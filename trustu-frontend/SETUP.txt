TrustU Frontend — Setup & Running Guide
========================================

REQUIREMENTS
------------
- Node.js v18 or higher  (check with: node -v)
- npm v9 or higher       (check with: npm -v)


INSTALL DEPENDENCIES
--------------------
Run this once after cloning the repo:

    npm install


ENVIRONMENT
-----------
The dev server proxies all /api requests to the backend automatically,
so you don't need to set any environment variables to get started.

The backend URL is configured in vite.config.ts:
    target: 'https://rosybrown-goat-692345.hostingersite.com'

If you need to point to a different backend, create a .env.local file:

    VITE_API_BASE_URL=https://your-backend-url.com

(This file is git-ignored — never commit it.)


RUNNING LOCALLY
---------------
Start the development server:

    npm run dev

Then open your browser at:

    http://localhost:3000

The page hot-reloads automatically when you save any file.


MOCK MODE (no backend needed)
------------------------------
You can run the entire app with dummy data — no backend connection required.
This is useful for quickly checking UI without needing valid credentials.

Enable:
    1. Open the app in your browser (npm run dev)
    2. Open DevTools console and run:
           localStorage.setItem('trustu_mock', 'true')
    3. Refresh the page
    4. Log in with any email and password — it will succeed

Disable:
    1. In DevTools console run:
           localStorage.removeItem('trustu_mock')
    2. Refresh the page

-----------------

AUTH FLOW
---------
1. Visit / → redirects to /auth/login
2. Enter credentials → if account doesn't exist, redirects to /auth/register
3. Register is a 2-step form:
     Step 1 — Name, email, password, designation, gender, phone, institute
     Step 2 — Home state and current state (backend auto-assigns community)
4. After login or register, syncProfile() is called to fetch full user data
   (community name, community ID, etc.) and store it in memory + localStorage
